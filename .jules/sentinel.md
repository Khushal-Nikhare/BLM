## 2024-05-26 - [ES Modules and dotenv Environment Variable Evaluation]
**Vulnerability:** The JWT_SECRET environment variable was being assigned at the top level of ES module files (`backend/middleware/auth.js` and `backend/routes/auth.js`) using a fallback to an insecure 'default_secret'. When index.js imports these modules, they are evaluated *before* `dotenv.config()` is called, causing `process.env.JWT_SECRET` to be undefined at that moment, falling back to the insecure default regardless of `.env` contents.
**Learning:** Due to how ES module imports are hoisted and evaluated in Node.js, environment variables loaded via `dotenv` in the entry point (`index.js`) are not available at the top level of imported modules. Relying on this pattern and using fallback secrets creates a critical vulnerability where an application appears correctly configured but is actually using a hardcoded, weak default secret.
**Prevention:** Always evaluate critical environment variables (like secrets and API keys) dynamically within functions where they are used, or ensure `dotenv` is configured via CLI arguments (e.g., `node -r dotenv/config`) or at the very top of a distinct initialization file before any application modules are imported. Additionally, fail securely by returning a 500 error instead of using insecure fallbacks.
