## 2024-05-09 - [Dynamic Evaluation of Environment Variables for Secrets]
**Vulnerability:** The `JWT_SECRET` environment variable was evaluated at the module scope using an insecure fallback `process.env.JWT_SECRET || 'default_secret'` during import. Due to ES Module load order, `process.env` might be empty prior to `dotenv.config()` executing in the application's entry point, which caused the application to silently fall back to using the hardcoded secret `'default_secret'`.
**Learning:** ES modules evaluate their dependencies before the entry file (e.g., `index.js`) executes synchronous code like `dotenv.config()`. Therefore, module-level environment variable evaluation in dependencies may capture undefined states, resulting in silent and dangerous reliance on fallbacks if not guarded properly.
**Prevention:** Always evaluate critical security environment variables dynamically at runtime within functions (e.g., inside route handlers or middleware) rather than at module load time. Additionally, implement strict guard clauses to explicitly throw an error or return a 500 Internal Server Error when missing critical secrets, instead of relying on insecure fallbacks.
