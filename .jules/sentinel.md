## 2024-05-18 - [CRITICAL] Prevent Hardcoded Secrets Fallback for JWT
**Vulnerability:** The application was using a hardcoded default JWT secret (`'default_secret'`) when the `JWT_SECRET` environment variable was missing. This is a severe security risk, as any attacker could generate valid JWT tokens with the `'default_secret'` and bypass authentication, or even escalate privileges (e.g., generating tokens with `{ role: 'ADMIN' }`).
**Learning:** This existed because of a common but dangerous pattern where developers add a fallback secret to make local development easier without requiring `.env` configuration. The fallback was present in both `backend/middleware/auth.js` and `backend/routes/auth.js`.
**Prevention:** Always enforce that sensitive environment variables like `JWT_SECRET` are explicitly set. If they are missing, the application should fail securely (e.g., by logging a fatal error and exiting with `process.exit(1)`) rather than falling back to an insecure default. This guarantees that production environments cannot accidentally run with known, weak secrets.
