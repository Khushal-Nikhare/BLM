## 2024-05-16 - Hardcoded Default JWT Secret Fallback due to ES Module Hoisting
**Vulnerability:** JWT tokens were signed and verified using a hardcoded default secret (`'default_secret'`) because the `process.env.JWT_SECRET` evaluation occurred at module load time before `dotenv.config()` was called in `index.js`.
**Learning:** In Node.js with ES Modules, `import` statements are hoisted and executed before the rest of the code in the file. Thus, environment variables loaded via `dotenv.config()` in the main entry point are not yet available when imported modules are initialized, causing top-level `process.env` evaluations in those modules to return `undefined` and fall back to hardcoded defaults.
**Prevention:** Evaluate critical environment variables dynamically at runtime (e.g., inside route handlers or middleware functions, or by using a getter function) rather than at the top level of the module. Ensure the application fails securely (e.g., 500 error) if a critical secret is missing, rather than falling back to an insecure default.
